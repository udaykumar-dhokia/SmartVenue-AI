# SmartVenue AI

AI-powered venue intelligence platform for the Narendra Modi Stadium, Ahmedabad. Real-time crowd management, smart routing, and personalized attendee experiences during IPL cricket matches.

## Architecture

The application is a Next.js (App Router) web application with two primary views:

### Admin Command Center

Full-screen operations dashboard for venue staff:

- **Interactive Stadium Map**: Real `map.svg` of the Narendra Modi Stadium with all seating sections (Stands A-R, Adani Pavilions, Premium Suites 4th/5th Floor, Banquet Seats). Each section is dynamically colored based on real-time crowd density (green to red gradient). Hover for zone tooltips, click for detailed drill-down.
- **AI Agent Feed**: Live stream of autonomous agent decisions generated by Google GenAI (Gemini 2.5 Flash). Five specialized agents (Crowd, Concession, Safety, Routing, Personalization) analyze venue state and generate actionable insights.
- **Stats Panel**: KPI cards showing attendance, average wait time, crowd safety score, flow efficiency, and active alert count with animated progress bars.
- **Alerts Panel**: Active alerts with severity-based color coding (critical/warning/info) and acknowledge actions.
- **Concession Panel**: Real-time outlet status showing wait times, items sold, revenue, and stock levels.
- **Zone Detail**: Slide-in panel on zone selection showing occupancy, density breakdown, temperature, incidents, section listing, and action buttons (Send Alert, Reroute Traffic).

### Attendee Companion

Mobile phone-framed interface simulating the attendee experience:

- **Stadium Map**: Simplified venue map highlighting the attendee's seat section with density overlays.
- **Smart Nudges**: AI-generated contextual suggestions powered by GenAI (restroom timing, food recommendations, exit routing, match event awareness).
- **Food Pre-Order**: Full menu with cart functionality, pricing in INR, and nearest outlet recommendation based on wait times.
- **Digital Ticket**: GT vs RCB match ticket card with seat details (Section J, Row F, Seat 24), QR code placeholder, gate assignment, and match metadata.
- **AI Chat Assistant**: Natural language chat interface powered by Gemini for venue queries (restroom locations, food queues, exit routes, match updates).

## Data Model

The stadium SVG contains groups identified by `data-category` attributes that map to the `VenueZone` data model:

- **22 zones** covering upper tier (J, K, L, M, N, P, Q, R), lower tier (A-H), pavilions (Left, Center, Right), suites (4th Floor 301-335, 5th Floor 401-438), and special areas (Banquet Seats)
- **8 concession outlets** with realistic Indian food items and INR pricing
- **Real-time simulation** updates crowd density, wait times, temperature, and concession metrics every 4 seconds

## Environment Variables

Create a `.env.local` file in the `app/` directory:

```env
GEMINI_API_KEY=your_gemini_api_key_here
```

Get your API key from [Google AI Studio](https://aistudio.google.com/).

Without the API key, the app falls back to pre-scripted agent messages, nudges, and chat responses.

## API Routes

### POST /api/agent

Server-side GenAI endpoint. Accepts:

```json
{
  "action": "thought" | "chat" | "nudge",
  "payload": {
    "context": "...",
    "question": "..."
  }
}
```

Returns:

```json
{
  "text": "Agent response..."
}
```

## Simulation Engine

The app uses a client-side simulation engine (`lib/simulation.ts`) that generates:

- Crowd density fluctuations per zone (every 4 seconds)
- Concession wait time and stock level changes
- Safety alerts and congestion warnings (every 15 seconds)
- Match score progression and events
- Smart nudge generation (every 18 seconds)
